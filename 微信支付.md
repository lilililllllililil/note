# 微信支付流程
 
```mermaid

sequenceDiagram
    participant phone as 用户手机(微信/支付宝浏览器)
    actor user as 用户
    participant pc as pc网页
    participant order as 订单服务
    participant pay as 支付服务
    user->>pc: 刷新二维码
    pc->>order: 调用接口创建预订单（将订单写到缓存）
    Note over order : 计算订单金额，将订单存入缓存
    order->>pc: 返回预订单缓存的 redis key
    user->>pc: 使用微信扫描二维码
    pc-->>phone: 打开微信扫码
    phone->>order: 手机上打开二维码中的链接，提交redis key拉起宝微支付
    Note over order: 将缓存中的订单存储到数据库
    Note over order: 识别接口发起请求方是在微信内置浏览器还是支付宝内置浏览器
    order->>pay: 请求支付服务，获取宝微支付配置
    pay->>order: 返回微信/支付宝需要的配置
    order->> phone: 返回支付配置到微信/支付宝浏览器
    Note over phone: 拉起微信支付
    user->>phone:输入支付密码
    Note over phone: 支付成功订单完成
