sequenceDiagram
    participant phone as 用户手机(微信/支付宝浏览器)
    actor user as 用户
    participant pc as pc网页
    participant order as 订单服务
    participant wx as 微信服务端
        participant pay as 支付服务
    user->>pc: 刷新二维码
    pc->>order: 调用接口创建预订单（将订单写到缓存）
    Note over order : 计算订单金额，将订单存入缓存
    order->>pc: 返回预订单缓存的 redis key
    user->>pc: 使用微信扫描二维码
    pc-->>phone: 打开微信扫码
    phone->>order: 手机上打开二维码中的链接，提交redis key拉起宝微支付， /toPay
    Note over order: 检查/toPay接口参数中有没有微信授权code
    order->>wx: 没有授权code，重定向到https://open.weixin.qq.com/connect/oauth2/authorize?appid=%s&redirect_uri=%s&response_type=code&scope=snsapi_base&state=STATE#wechat_redirect
    wx->>pay: 携带授权重定向到/oauth/getCode
    pay->>order: 携带授权code重定向到/topay接口
    Note over order: 验证是否有授权code，有就不再申请
    order->>pay: 请求支付服务创建支付订单，并且获取拉起微信支付需要的参数
    Note over pay: 创建支付订单
    Note over pay: 扣款虚余额、使用优惠券
    Note over pay: 如果需要实付rmb，根据参数检查是：微信还是支付宝支付
    Note over pay: 调用第三方接口，创建第三方订单，返回拉起支付的参数
    pay->>order:返回拉起支付需要的参数
    Note over order: 将支付参数写到mv里面，mv会构建一个html网页
    order ->> phone: 返回html页面
    Note over phone: html网页中通过js调起微信支付
    phone->>user:需要用户输入支付密码
